Conceptual Model



Entities & Attributes


User (strong entity)

  • userId (primary key)
  • userHash (user password)
  • userActivationToken (for account verification)
  • userLocation
  • userEmail
  • userPhoneNumber

Post (strong entity)

  • postId (primary key)
  • postStatus
  • postManageStateButtons (delete, display, renew, etc)
  • postTitle
  • postAreaAndCategory
  • postDate
  • postExpiration

User Post Status (weak entity)

  • userId (foreign key)
  • postId (foreign key)
  • postStatus

Relations

  • One User can have many Posts (1 to n)
  • Each Post can be from only one User (n to 1)
  • Each User Post can have multiple User Post Status states
